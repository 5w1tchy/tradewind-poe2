**Golden rule: `src/core` stays pure** â€” no Electron, no DOM, no `window`. It is

## The core loop

1. Native keyboard hook sees `Ctrl+D` while PoE2 is focused; the keypress is swallowed.
2. `main/` synthesizes the game's advanced copy (`Ctrl+Alt+C`), polls the clipboard, restores it.
3. `core/parser` â†’ `ParsedItem`; `core/stats-db` matcher maps mod lines to stat IDs; `core/query/prepare` builds a `PreparedQuery`.
4. Main sends `tw:item` (`ItemPayload`) to the renderer, which shows the popup and auto-searches (uniques/gems/currency/white bases) or arms the Search button (rares).
5. Renderer calls `search()` â†’ main runs `POST /search` + `GET /fetch` through the single rate-limit queue; `core/pricing` produces the estimate; results render.

## Conventions & gotchas

- **React, not Vue.** `PLAN.md` predates the migration and still says "Vue 3 + Tailwind" â€” ignore that; the renderer is React 19 with **CSS Modules** (`*.module.css`). There is **no Tailwind**. Generic class names (`.name`, `.row`, `.tier`) collide across components, which is why styles are scoped modules; only the global theme primitives (`tw-label`, `tw-btn`, `tw-menu`) in `src/renderer/src/styles/theme.css` are plain global class strings.
- **No `<StrictMode>`.** The preload appends IPC listeners with no unsubscribe, so the dev double-invoke would deliver every item twice. `onItem`/`onHide` are subscribed exactly once in `App.tsx`'s mount effect â€” keep it that way.
- **IPC channels** (`tw:*`) are defined in `preload/index.ts` and handled in `main/index.ts`; the typed surface is `src/shared/ipc.ts`. Add new calls in all three.
- **Popup is a fixed-size, resizable box** (`App.tsx`, issue #35): the popup `<div>` has an explicit width/height (not content-sized), placed **top-middle** (horizontally centered, ~8% down) and **draggable** from the header bar. Because the frame size is fixed, **content changes never resize it** â€” switching Priceâ†”Craft or loading results leaves the frame put; each tab panel fills the frame via the flex-column `.body` and **scrolls internally** (the stats list, the results list, the essence list own their own overflow). Drag the **bottom-right corner handle** to resize the whole popup; on the Price tab, drag the **grip above the results list** to set its height while the stats list flex-fills the rest. Both sizes persist to `config.json` (`popupSize` via `tw:set-popup-size`, `resultsHeight` via `tw:set-results-height`) and are restored on the next check (sent back in `ItemPayload`). Anchor resize gestures to the rendered `size`/`resultsHeight` state, **not** `el.offsetWidth` (which includes padding+border and snaps the first drag tick). `config.ts`'s `sanitize()` coerces/clamps every loaded field, so a malformed or schema-changed `config.json` heals to defaults instead of leaking `NaN`.
- **Overlay popup behavior** (`App.tsx` + `main/index.ts`): the popup mimics PoE2's item tooltip (issue #32). The top-right header button is one **morphing control** (`setPinned` â†’ `tw:set-pinned`): a **pin** glyph while unpinned (click to pin), an **âś•** while pinned (click to close â€” there is no separate unpin). **Unpinned** (default): an outside click or Esc closes it. **Pinned**: outside clicks are ignored; only the âś• or Esc close it, and a **fresh price check updates the popup in place and keeps the pin** â€” pin state persists across searches and resets to unpinned **only when the popup closes** (`hidePopup` for Esc/outside; the null `tw:popup-rect` for the âś•). The renderer reports its rect via `setPopupRect`; the main process hit-tests the cursor against it to toggle click-through (`setIgnoreMouseEvents`) and, on a `mousedown` outside it (observed via uiohook), auto-hides when unpinned. Report the rect synchronously from placement; never from a stale `pos` state (a `{0,0}` ghost report makes main treat the popup as off in the corner).
- **Software rendering on purpose.** `main/index.ts` calls `app.disableHardwareAcceleration()` at startup. On machines where the GPU is present but only half-engaged for Chromium (it shows up yet `app.getGPUFeatureStatus().gpu_compositing` is a `disabled_software`/WARP fallback), letting Electron try the GPU and limp along in that path makes the popup lag hard (jerky drag, multi-second hover repaints). The overlay is a tiny, mostly static UI that doesn't need the GPU, so forcing clean software rendering is smooth everywhere and dodges that whole fallback class. Don't "optimize" this away.
- **Input has three tiers** (`main/index.ts`, `keyhook.ts`, `input.ts`): native `WH_KEYBOARD_LL` addon (`tradewind-keyhook` via koffi, best â€” blocks the key before the game's key-state table) â†’ `globalShortcut` â†’ `uiohook-napi` observe-only. The native tier prevents `Ctrl+D` leaking as a WASD-style game input.
- **Tests are fixture-driven.** `fixtures/` holds real in-game `Ctrl+Alt+C` copies. âš ď¸Ź `npm run fixtures` (`scripts/split-fixtures.mjs`) **wipes any manual fixture not present in `fixtures/inbox.txt`** â€” add new samples to `inbox.txt`, don't hand-place fixture files.
- **Tray is the only chrome.** The sole window is the click-through overlay, hidden whenever PoE2 isn't focused â€” so the system-tray icon (`main/tray.ts`) is the only way to quit (Quit) and to trigger a manual update check (Check for updatesâ€¦, wired to `checkForUpdatesManually` in `updater.ts`, which â€” unlike the silent startup/poll checks â€” reports its result with a native dialog and offers an immediate restart-to-install). Keep the `tray` reference alive in `index.ts` or the GC reaps it from the notification area. The app/installer icon is `build/icon.png` (the compass logo, already cropped tight to its content), picked up automatically by electron-builder. `build/` isn't bundled into the asar, so the tray can't read it at runtime â€” `scripts/gen-tray.mjs` (a dependency-free PNG decode â†’ 32Ă—32 downscale â†’ encode) prints the base64 to paste into `tray.ts`'s `ICON_PNG`, so dev and the packaged app load identical bytes. New logo â†’ drop it in as `build/icon.png`, run `gen-tray.mjs`.
- **Startup splash** (`main/splash.ts`): a centered, frameless, transparent, always-on-top window showing `build/splash.png`, created first thing in `whenReady` and dismissed once the four startup data fetches (`statsReady`/`exchangeReady`/`baseTypesReady`/`leaguesReady`) settle â€” but with a `SPLASH_MIN_MS` (2.5s) floor so it never just flashes and a `SPLASH_MAX_MS` (10s) cap so a hung network can't trap it. It fades in/out via a CSS opacity transition on the image (not window opacity, which is unreliable on transparent windows); `createSplashWindow` returns a `Splash` whose idempotent `close()` fades out then destroys. The image is bundled via electron-vite's `?asset` import (copied into `out/main/chunks`, packed into the asar â€” see the `*?asset` decl in `main/env.d.ts`) and inlined as a data URL, so unlike the tray it needs no base64 regen step. The other `build/*.png` are referenced the same way only by electron-builder, not at runtime.
- **Auto-update** (`main/updater.ts`, `electron-builder.yml`): GitHub-releases feed (provider config â†’ baked `app-update.yml`; no URL in code). The **startup** check downloads and then silently installs-and-relaunches (`quitAndInstall(true, true)`) â€” gated by a one-shot `autoInstallOnStartup` flag so a later background-poll find can't yank a live game; poll-found updates wait for `autoInstallOnAppQuit` or the tray restart. Silent installs require the NSIS **`oneClick: true`** installer (no wizard, no UAC, per-user) â€” `oneClick: false` shows the assisted wizard even for updates. Note: detection lag after publishing is GitHub's CDN caching the release feed, not us; restarts don't bust it.
- **ToS posture:** clipboard + overlay only â€” no memory reading, no injection, one game action per keypress. All GGG/poe2scout calls go through the single rate-limit queue honoring `X-Rate-Limit-*`/`Retry-After`. Honest `User-Agent`. Requires **windowed-fullscreen** (overlays can't draw over exclusive fullscreen).

## Git

Work on a feature branch, push it, and open a PR against `main` â€” `main` is
protected, no direct pushes. End commit messages with the `Co-Authored-By`
trailer.

Releases are tag-driven (push a `vX.Y.Z` tag â†’ `.github/workflows/release.yml`).
The **`/release` skill** (`.claude/skills/release/SKILL.md`, committed so every
collaborator has it) cuts a release in two phases around the PR-merge gate:
Phase A infers the next version from commits since the last tag, bumps
`package.json`, prepends the user-facing entry to `CHANGELOG.md`, and opens the
release PR; Phase B (after merge) tags, lets CI publish, then sets the GitHub
Release notes from the `CHANGELOG.md` entry. `CHANGELOG.md` is the single source
of truth for the polished notes.
