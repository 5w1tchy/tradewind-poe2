# Tradewind — PoE2 price-check overlay

One hotkey on a hovered Path of Exile 2 item → an overlay popup with a price
estimate and live trade listings. Electron + TypeScript + React.

> **Keep this file current.** If a change alters anything described here —
> commands, the process boundary, the `src/core` purity rule, the IPC surface,
> overlay/popup behavior, the input tiers, or any convention/gotcha below —
> update CLAUDE.md in the **same** change. A stale CLAUDE.md is worse than none.

## Commands

```
npm run dev        # electron-vite dev — launches the overlay, watches for the PoE2 window, HMR
npm run build      # electron-vite build → out/
npm start          # electron-vite preview (built app)
npm test           # vitest run — parser/matcher/query/pricing/craft unit tests
npm run fixtures   # re-split fixtures/inbox.txt into per-item test fixtures
npx tsc -b         # typecheck (project refs: tsconfig.node.json + tsconfig.web.json)
```

Browser UI preview without the game or Electron:
`npx vite --config vite.preview.config.ts` then open `/preview.html` (mocks
`window.tradewind` with a canned item — see `src/renderer/src/preview.ts`).

## Layout & the process boundary

```
src/
├─ core/      Pure TypeScript, NO electron imports. Unit-tested. The brain.
│   ├─ parser/    clipboard item text → ParsedItem (splits on "--------" sections)
│   ├─ stats-db/  live-fetched /api/trade2/data → runtime matcher generation (# placeholders → regex)
│   ├─ query/     ParsedItem + filters → PreparedQuery → trade2 search body (prepare.ts, build.ts)
│   ├─ pricing/   currency normalization, outlier rejection, estimate range + confidence
│   ├─ craft/     essence crafting helper (essences.json generated by scripts/gen-essences.mjs)
│   └─ trade/     SearchOutcome / listing types
├─ main/      Electron main process. Win32 window tracking, global hotkey, clipboard, API gateway.
├─ preload/   contextBridge — exposes window.tradewind (see src/shared/ipc.ts for the surface)
├─ renderer/  React 19 (.tsx + CSS Modules). The popup UI.
└─ shared/    ipc.ts — the typed IPC contract shared by main, preload, renderer
```

**Golden rule: `src/core` stays pure** — no Electron, no DOM, no `window`. It is
imported by both the main process and the renderer and must stay framework- and
runtime-agnostic. All side effects (network, clipboard, windows) live in `main/`.

## The core loop

1. Native keyboard hook sees `Ctrl+D` while PoE2 is focused; the keypress is swallowed.
2. `main/` synthesizes the game's advanced copy (`Ctrl+Alt+C`), polls the clipboard, restores it.
3. `core/parser` → `ParsedItem`; `core/stats-db` matcher maps mod lines to stat IDs; `core/query/prepare` builds a `PreparedQuery`.
4. Main sends `tw:item` (`ItemPayload`) to the renderer, which shows the popup and auto-searches (uniques/gems/currency/white bases) or arms the Search button (rares).
5. Renderer calls `search()` → main runs `POST /search` + `GET /fetch` through the single rate-limit queue; `core/pricing` produces the estimate; results render.

## Conventions & gotchas

- **React, not Vue.** `PLAN.md` predates the migration and still says "Vue 3 + Tailwind" — ignore that; the renderer is React 19 with **CSS Modules** (`*.module.css`). There is **no Tailwind**. Generic class names (`.name`, `.row`, `.tier`) collide across components, which is why styles are scoped modules; only the global theme primitives (`tw-label`, `tw-btn`, `tw-menu`) in `src/renderer/src/styles/theme.css` are plain global class strings.
- **No `<StrictMode>`.** The preload appends IPC listeners with no unsubscribe, so the dev double-invoke would deliver every item twice. `onItem`/`onHide` are subscribed exactly once in `App.tsx`'s mount effect — keep it that way.
- **IPC channels** (`tw:*`) are defined in `preload/index.ts` and handled in `main/index.ts`; the typed surface is `src/shared/ipc.ts`. Add new calls in all three.
- **Overlay popup behavior** (`App.tsx` + `main/index.ts`): the popup is centered, **stays open until dismissed** (its ✕ button or Esc), and is **draggable** from the header bar. The renderer reports its rect via `setPopupRect`; the main process hit-tests the cursor against it *only* to toggle click-through (`setIgnoreMouseEvents`) — it never auto-hides. Report the rect synchronously from placement; never from a stale `pos` state (a `{0,0}` ghost report makes main treat the popup as off in the corner).
- **Input has three tiers** (`main/index.ts`, `keyhook.ts`, `input.ts`): native `WH_KEYBOARD_LL` addon (`tradewind-keyhook` via koffi, best — blocks the key before the game's key-state table) → `globalShortcut` → `uiohook-napi` observe-only. The native tier prevents `Ctrl+D` leaking as a WASD-style game input.
- **Tests are fixture-driven.** `fixtures/` holds real in-game `Ctrl+Alt+C` copies. ⚠️ `npm run fixtures` (`scripts/split-fixtures.mjs`) **wipes any manual fixture not present in `fixtures/inbox.txt`** — add new samples to `inbox.txt`, don't hand-place fixture files.
- **ToS posture:** clipboard + overlay only — no memory reading, no injection, one game action per keypress. All GGG/poe2scout calls go through the single rate-limit queue honoring `X-Rate-Limit-*`/`Retry-After`. Honest `User-Agent`. Requires **windowed-fullscreen** (overlays can't draw over exclusive fullscreen).
