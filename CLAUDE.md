# Tradewind ‚Äî PoE2 price-check overlay

One hotkey on a hovered Path of Exile 2 item ‚Üí an overlay popup with a price
estimate and live trade listings. Electron + TypeScript + React.

> **Keep this file current.** If a change alters anything described here ‚Äî

> overlay/popup behavior, the input tiers, or any convention/gotcha below ‚Äî

## Commands

```
npm run dev        # electron-vite dev ‚Äî launches the overlay, watches for the PoE2 window, HMR
npm run build      # electron-vite build ‚Üí out/
npm start          # electron-vite preview (built app)
npm test           # vitest run ‚Äî parser/matcher/query/pricing/craft unit tests
npm run fixtures   # re-split fixtures/inbox.txt into per-item test fixtures
npx tsc -b         # typecheck (project refs: tsconfig.node.json + tsconfig.web.json)
```

Browser UI preview without the game or Electron:
`npx vite --config vite.preview.config.ts` then open `/preview.html` (mocks
`window.tradewind` with a canned item ‚Äî see `src/renderer/src/preview.ts`).

## Layout & the process boundary

```
src/
‚îú‚îÄ core/      Pure TypeScript, NO electron imports. Unit-tested. The brain.
‚îÇ   ‚îú‚îÄ parser/    clipboard item text ‚Üí ParsedItem (splits on "--------" sections)
‚îÇ   ‚îú‚îÄ stats-db/  live-fetched /api/trade2/data ‚Üí runtime matcher generation (# placeholders ‚Üí regex)
‚îÇ   ‚îú‚îÄ query/     ParsedItem + filters ‚Üí PreparedQuery ‚Üí trade2 search body (prepare.ts, build.ts)
‚îÇ   ‚îú‚îÄ pricing/   currency normalization, outlier rejection, estimate range + confidence
‚îÇ   ‚îú‚îÄ craft/     essence crafting helper (essences.json generated by scripts/gen-essences.mjs)
‚îÇ   ‚îî‚îÄ trade/     SearchOutcome / listing types
‚îú‚îÄ main/      Electron main process. Win32 window tracking, global hotkey, clipboard, API gateway.
‚îú‚îÄ preload/   contextBridge ‚Äî exposes window.tradewind (see src/shared/ipc.ts for the surface)
‚îú‚îÄ renderer/  React 19 (.tsx + CSS Modules). The popup UI.
‚îî‚îÄ shared/    ipc.ts ‚Äî the typed IPC contract shared by main, preload, renderer
```

**Golden rule: `src/core` stays pure** ‚Äî no Electron, no DOM, no `window`. It is
imported by both the main process and the renderer and must stay framework- and
runtime-agnostic. All side effects (network, clipboard, windows) live in `main/`.

## The core loop

1. Native keyboard hook sees `Ctrl+D` while PoE2 is focused; the keypress is swallowed.
2. `main/` synthesizes the game's advanced copy (`Ctrl+Alt+C`), polls the clipboard, restores it.
3. `core/parser` ‚Üí `ParsedItem`; `core/stats-db` matcher maps mod lines to stat IDs; `core/query/prepare` builds a `PreparedQuery`.
4. Main sends `tw:item` (`ItemPayload`) to the renderer, which shows the popup and auto-searches (uniques/gems/currency/white bases) or arms the Search button (rares).
5. Renderer calls `search()` ‚Üí main runs `POST /search` + `GET /fetch` through the single rate-limit queue; `core/pricing` produces the estimate; results render.

## Conventions & gotchas

- **React, not Vue.** `PLAN.md` predates the migration and still says "Vue 3 + Tailwind" ‚Äî ignore that; the renderer is React 19 with **CSS Modules** (`*.module.css`). There is **no Tailwind**. Generic class names (`.name`, `.row`, `.tier`) collide across components, which is why styles are scoped modules; only the global theme primitives (`tw-label`, `tw-btn`, `tw-menu`) in `src/renderer/src/styles/theme.css` are plain global class strings.
- **No `<StrictMode>`.** The preload appends IPC listeners with no unsubscribe, so the dev double-invoke would deliver every item twice. `onItem`/`onHide` are subscribed exactly once in `App.tsx`'s mount effect ‚Äî keep it that way.
- **IPC channels** (`tw:*`) are defined in `preload/index.ts` and handled in `main/index.ts`; the typed surface is `src/shared/ipc.ts`. Add new calls in all three.
- **Overlay popup behavior** (`App.tsx` + `main/index.ts`): the popup is centered, **stays open until dismissed** (its ‚úï button or Esc), and is **draggable** from the header bar. The renderer reports its rect via `setPopupRect`; the main process hit-tests the cursor against it *only* to toggle click-through (`setIgnoreMouseEvents`) ‚Äî it never auto-hides. Report the rect synchronously from placement; never from a stale `pos` state (a `{0,0}` ghost report makes main treat the popup as off in the corner).
- **Input has three tiers** (`main/index.ts`, `keyhook.ts`, `input.ts`): native `WH_KEYBOARD_LL` addon (`tradewind-keyhook` via koffi, best ‚Äî blocks the key before the game's key-state table) ‚Üí `globalShortcut` ‚Üí `uiohook-napi` observe-only. The native tier prevents `Ctrl+D` leaking as a WASD-style game input.
- **Tests are fixture-driven.** `fixtures/` holds real in-game `Ctrl+Alt+C` copies. ‚öÝÔ∏è `npm run fixtures` (`scripts/split-fixtures.mjs`) **wipes any manual fixture not present in `fixtures/inbox.txt`** ‚Äî add new samples to `inbox.txt`, don't hand-place fixture files.
- **Tray is the only chrome.** The sole window is the click-through overlay, hidden whenever PoE2 isn't focused ‚Äî so the system-tray icon (`main/tray.ts`) is the only way to quit (Quit) and to trigger a manual update check (Check for updates‚Ä¶, wired to `checkForUpdatesManually` in `updater.ts`, which ‚Äî unlike the silent startup/poll checks ‚Äî reports its result with a native dialog and offers an immediate restart-to-install). Keep the `tray` reference alive in `index.ts` or the GC reaps it from the notification area. The app/installer icon is `build/icon.png` (the compass logo, already cropped tight to its content), picked up automatically by electron-builder. `build/` isn't bundled into the asar, so the tray can't read it at runtime ‚Äî `scripts/gen-tray.mjs` (a dependency-free PNG decode ‚Üí 32√ó32 downscale ‚Üí encode) prints the base64 to paste into `tray.ts`'s `ICON_PNG`, so dev and the packaged app load identical bytes. New logo ‚Üí drop it in as `build/icon.png`, run `gen-tray.mjs`.
- **Startup splash** (`main/splash.ts`): a centered, frameless, transparent, always-on-top window showing `build/splash.png`, created first thing in `whenReady` and dismissed once the four startup data fetches (`statsReady`/`exchangeReady`/`baseTypesReady`/`leaguesReady`) settle ‚Äî but with a `SPLASH_MIN_MS` (2.5s) floor so it never just flashes and a `SPLASH_MAX_MS` (10s) cap so a hung network can't trap it. It fades in/out via a CSS opacity transition on the image (not window opacity, which is unreliable on transparent windows); `createSplashWindow` returns a `Splash` whose idempotent `close()` fades out then destroys. The image is bundled via electron-vite's `?asset` import (copied into `out/main/chunks`, packed into the asar ‚Äî see the `*?asset` decl in `main/env.d.ts`) and inlined as a data URL, so unlike the tray it needs no base64 regen step. The other `build/*.png` are referenced the same way only by electron-builder, not at runtime.
- **Auto-update** (`main/updater.ts`, `electron-builder.yml`): GitHub-releases feed (provider config ‚Üí baked `app-update.yml`; no URL in code). The **startup** check downloads and then silently installs-and-relaunches (`quitAndInstall(true, true)`) ‚Äî gated by a one-shot `autoInstallOnStartup` flag so a later background-poll find can't yank a live game; poll-found updates wait for `autoInstallOnAppQuit` or the tray restart. Silent installs require the NSIS **`oneClick: true`** installer (no wizard, no UAC, per-user) ‚Äî `oneClick: false` shows the assisted wizard even for updates. Note: detection lag after publishing is GitHub's CDN caching the release feed, not us; restarts don't bust it.
- **ToS posture:** clipboard + overlay only ‚Äî no memory reading, no injection, one game action per keypress. All GGG/poe2scout calls go through the single rate-limit queue honoring `X-Rate-Limit-*`/`Retry-After`. Honest `User-Agent`. Requires **windowed-fullscreen** (overlays can't draw over exclusive fullscreen).
